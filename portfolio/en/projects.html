Workflow Generator

Live Demo GitHub

A web app that automatically generates diagrams and Word manuals from business process descriptions

Motivation

I noticed that visualizing business processes and managing them as documents was tedious and time-consuming at work.
I started development with the idea that a tool which automatically creates diagrams and manuals from text input could significantly reduce the documentation burden.

Key Features

AI Auto-Structuring

Claude API analyzes text-based process descriptions and automatically converts them into a structured format of departments, groups, processes, and steps.

SwimLane + LinearFlow

Renders SwimLane diagrams showing departmental roles at a glance and LinearFlow diagrams showing step-by-step processes as SVG.

Auto Word Manual Generation

Generates .docx documents with cover page, summary table, step-by-step descriptions, and images directly in the browser.

Drag & Drop Editing

Step reordering via @dnd-kit with parallel branching (colIndex) support for intuitive editing of complex processes.

Architecture

User Input
Text / Manual Entry
Netlify Functions
Claude API Proxy
React SPA
SVG Diagrams + Editor
Output
Diagram + Word .docx
LocalStorage (text) IndexedDB (images)

Problem Solving

Problem

When users describe a process as text, AI structures it — but real workflows often involve parallel flows like "A and B run simultaneously," which a simple sequential list couldn't represent

Solution

Assigned a colIndex (column number) to each step so that steps in the same column are automatically treated as parallel groups, and designed @dnd-kit drag-and-drop to intuitively edit order and branching

Code Review Pipeline

GitHub

A multi-agent system where 4 AI agents sequentially review and score code

Motivation

While rapidly iterating on AI automation projects, I felt the need to systematically manage security and performance checkpoints.
I developed this system with the idea that having specialized reviewers for security, performance, and code quality could prevent these issues proactively.

Key Features

4-Agent Sequential Analysis

Security, Performance, Code Quality, and Summary agents run in sequence, with each receiving the analysis results of previous agents as context.

Cross-Review

Subsequent agents provide agree/caution/disagree opinions on previous agents' suggestions, delivering multi-perspective reviews.

100-Point Scoring

Calculates scores based on Critical (-15), Warning (-5), Info (-1) criteria, and automatically suggests Top 3 fix priorities.

GitHub PR Integration

Enter a PR URL to automatically extract and review changed file diffs. Direct file upload is also supported.

Architecture

Input Code / PR URL / File
Agent 1 Security
Agent 2 Performance
Agent 3 Quality
Agent 4 Summary + Score
Output Report + Markdown

Each agent receives accumulated results from previous agents for cross-verification. Tool Use enforces JSON structure to minimize parsing failures.

Problem Solving

Problem

With 4 agents running sequentially, a single failure would halt the entire pipeline, causing all remaining analysis results to be lost

Solution

Isolated each agent with try-except, logging failures via state.add_error() and automatically proceeding to the next agent — a Partial Failure architecture

AI Consulting Agent

GitHub

A CLI consulting tool where 3 AIs simultaneously analyze and cross-verify MS automation solutions

Motivation

There were frequent requests for automation using the Microsoft ecosystem, but with so many options — Power Automate, Office Script, Power Query, etc. — it was difficult to determine which approach was most suitable. I also felt the need to systematically document user requests and solutions, which led me to build a tool that handles everything from requirement analysis to solution comparison and deliverable generation in one go.

Key Features

Quick / Deep Mode

Quick mode provides fast results via Claude-only analysis. Deep mode delivers cross-verified results through parallel analysis by Claude + Codex + Gemini.

Re-consulting System

Supports 3 types of re-consulting: condition change (A), solution change (B), and explanation request (C). Session state is file-managed so history is preserved.

ROI Prediction

Calculates ROI across 3 scenarios (conservative/neutral/aggressive) based on time spent before and after automation.

Multiple Deliverables

Automatically generates consulting reports (.txt), Excel reports (.xlsx), and Power Automate Flow design documents.

Architecture

Step 1 Requirement Parsing
Step 2 Scope Gate
Step 3 AI Analysis (3-AI)
Step 4 Risk 2-Pass
Step 5 Deliverable Generation

Validates solution support via MS Learn auto-search. Deprecated products are automatically excluded through a Blocklist.

Problem Solving

Problem

Response structures from Claude, Codex, and Gemini were all different, making comparison and integration impossible

Solution

Defined a common output schema, with a normalization retry on parse failure + automatic fallback to another AI when CLI call exceeds 120-second timeout

Notion Notes

GitHub

A web app that analyzes study materials with AI and selectively saves useful responses to Notion

Motivation

I wanted to organize what I learned through AI conversations and save only the useful responses directly to Notion. Manually copying AI responses and pasting them into Notion felt inefficient, so I built a tool that handles everything from file upload to analysis and saving in one place.

Key Features

Multi-File Upload

Drag and drop PDF, image, Word, and text files. Processes them via Gemini File API to save tokens.

Multi-AI Chat

Freely switch between Gemini and OpenAI models for multi-turn conversations. UI adapts automatically based on API key availability.

Selective Saving

Select only useful AI responses via checkboxes to save to Notion. Save just what you need, not everything.

Notion Hierarchy Integration

Supports Page → Database → Item hierarchy. Create new items or append content to existing ones.
